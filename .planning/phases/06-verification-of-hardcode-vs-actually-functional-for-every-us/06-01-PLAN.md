  <action>
    Create `scripts/hardcodeScanner.mjs` — a standalone Node.js script (ESM, .mjs extension) that scans the codebase for hardcoded patterns. Use only Node.js built-in modules (fs, path) — no npm dependencies.

    The scanner MUST check these categories (per D-04):

    ### Category A: Hardcoded Firebase Auth UID patterns (D-04a)
    - Search all `.ts`, `.tsx` files under `src/` for any string matching Firebase Auth UID pattern (`u_001`, `u_002`, etc.)
    - Search for any remaining `CURRENT_USER_ID` references
    - Exclude matches inside `.planning/` directory
    - Exclude matches inside `node_modules/`

    ### Category B: Mock data imports from src/constants/data (D-04b, D-06)
    - Search all `.ts`, `.tsx` files under `src/` for imports from `@/src/constants/data` or `../constants/data`
    - Report each file that imports from data.ts and what it imports
    - Also scan for usage of these exported symbols: `USER`, `PLANTS`, `ACTIVITIES`, `ACHIEVEMENTS`, `DAILY_MISSIONS`, `DAILY_TIP`, `PLANT_OF_THE_DAY`, `RECENT_ACHIEVEMENT`
    - Check which exported functions from data.ts are still called: `getLevelProgress`, `getLastIdentifiedPlant`, `getAccountAge`

    ### Category C: Hardcoded Firebase config keys (D-04c, D-05)
    - Check `src/config/firebase.js` for hardcoded values in the `firebaseConfig` object
    - Report: apiKey, authDomain, projectId, storageBucket, messagingSenderId, appId
    - Also check for any other `.ts`/`.tsx` files that duplicate these keys (hardcoded in other files)

    ### Category D: "Próximamente..." placeholder screens (D-06)
    - Search all `.ts`, `.tsx` files under `src/` for the string "Próximamente" or "próximamente"
    - Report file, line number, and surrounding context

    ### Category E: Hardcoded fallback URLs
    - Search for hardcoded URLs (https://...) in source code that are not in config files
    - Report the `EXPO_PUBLIC_API_URL` fallback specifically

    ### Scanner Output Format
    The scanner must output a JSON report to stdout with this structure:
    ```json
    {
      "scannedAt": "2026-06-03T00:00:00.000Z",
      "summary": {
        "totalFilesScanned": 123,
        "totalFindings": 8,
        "byCategory": {
          "AUTH_UID": 0,
          "MOCK_DATA": 2,
          "FIREBASE_CONFIG": 1,
          "PLACEHOLDER": 1,
          "HARDCODED_URL": 1
        }
      },
      "findings": [
        {
          "id": "F-01",
          "category": "FIREBASE_CONFIG",
          "severity": "HIGH",
          "file": "src/config/firebase.js",
          "line": 7,
          "description": "Firebase API keys hardcoded in source",
          "recommendation": "Move to .env using EXPO_PUBLIC_ prefix"
        }
      ]
    }
    ```

    ### Script Usage
    Add a npm script in `package.json`:
    ```json
    "scripts": {
      ...existing,
      "scan:hardcode": "node scripts/hardcodeScanner.mjs"
    }
    ```

    ### Scanner Implementation Details
    - Use `fs.readdirSync` with recursion to walk `src/` directory
    - Use `fs.readFileSync` with `utf-8` encoding to read each file
    - Use regex matching for pattern detection
    - Ignore: `node_modules/`, `.planning/`, `__tests__/` (test files are expected to have mocks), `.expo/`
    - For .ts/.tsx files, strip comments before scanning to avoid false positives from comment references
    - Implement a `--json` flag that outputs only the JSON report (for programmatic use)
    - Implement a `--markdown` flag that outputs a human-readable markdown report (default: both JSON to stdout + markdown summary)

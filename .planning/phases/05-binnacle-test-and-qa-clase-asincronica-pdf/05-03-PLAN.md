    # Parte 2 — Testing/QA
    ## Investigación de herramientas
    **Herramienta 1: Jest + React Native Testing Library**
    - Tipo: Unit/Component
    - Facilidad en Expo: Alta (preset jest-expo)
    - Pros: estándar, amplia comunidad, integra bien con RN
    - Contras: configuración inicial, mocks para módulos nativos

    **Herramienta 2: Detox**
    - Tipo: E2E
    - Facilidad en Expo: Media/Baja (requiere configuración nativa)
    - Pros: E2E real, flujos completos
    - Contras: setup pesado en Expo managed

    **Herramienta 3: Maestro**
    - Tipo: E2E
    - Facilidad en Expo: Media (configuración externa)
    - Pros: sintaxis simple, rápido para smoke tests
    - Contras: integración limitada con Expo

    **Conclusión:** Se recomienda Jest + RTL por facilidad en Expo y cobertura de pruebas unitarias y de componentes.

    ## Implementación de pruebas básicas
    **Prueba 1 (Unit): Validación de datos de planta**
    - Descripción: valida nombre común, nombre científico, días de riego y luz.
    - Código: `src/lib/__tests__/plantValidation.test.ts`
    - Resultado: `npm test -- --runTestsByPath src/lib/__tests__/plantValidation.test.ts` (captura terminal incluida)

    **Prueba 2 (Component): LoginScreen error formulario vacío**
    - Descripción: al presionar "Entrar" con campos vacíos, muestra error.
    - Código: `src/screens/auth/__tests__/loginScreen.test.tsx`
    - Resultado: `npm test -- --runTestsByPath src/screens/auth/__tests__/loginScreen.test.tsx` (captura terminal incluida)

    ## Reflexión

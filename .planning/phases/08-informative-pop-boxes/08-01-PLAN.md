  <action>
    Create `src/services/popupService.ts` with three key namespacing strategies and four exported functions.

    AsyncStorage key format:
    - Per-session dismissals: `@plantasmon/popup_session:{popupKey}`
    - Permanent dismissals: `@plantasmon/popup_forever:{popupKey}`

    Functions to export:

    1. `isPopupDismissed(popupKey: string): Promise<boolean>`
       - Check permanent key first: `await AsyncStorage.getItem(foreverKey(popupKey))`
       - If found → return true (dismissed forever)
       - Check session key: `await AsyncStorage.getItem(sessionKey(popupKey))`
       - If found → return true (dismissed this session)
       - Neither found → return false

    2. `dismissPopup(popupKey: string): Promise<void>`
       - Store `"true"` under session key: `await AsyncStorage.setItem(sessionKey(popupKey), "true")`
       - Per D-04: "Popups shown every session until user presses 'Entendido' or 'No mostrar de nuevo'"
       - So this only sets session flag, not permanent

    3. `dismissForever(popupKey: string): Promise<void>`
       - Store `"true"` under BOTH session and forever keys
       - `await AsyncStorage.setItem(sessionKey(popupKey), "true")`
       - `await AsyncStorage.setItem(foreverKey(popupKey), "true")`
       - Per D-05: "Once dismissed, never shows again for that feature"

    4. `resetPopup(popupKey: string): Promise<void>`
       - Remove both session and forever keys
       - `await AsyncStorage.removeItem(sessionKey(popupKey))`
       - `await AsyncStorage.removeItem(foreverKey(popupKey))`
       - Useful for testing or admin reset

    Helper constants (not exported):
    ```typescript
    const STORAGE_PREFIX = "@plantasmon/popup_";
    const SESSION_SUFFIX = "session";
    const FOREVER_SUFFIX = "forever";
    const sessionKey = (key: string) => `${STORAGE_PREFIX}${SESSION_SUFFIX}:${key}`;
    const foreverKey = (key: string) => `${STORAGE_PREFIX}${FOREVER_SUFFIX}:${key}`;
    ```

    Follow existing codebase patterns:
    - `import AsyncStorage from '@react-native-async-storage/async-storage'` (same as offlineStorage.ts)
    - Named exports
    - try/catch with console.error
    - Spanish comments are acceptable (matches app locale — Spanish-first codebase)

    ```typescript
    import { useState, useEffect, useCallback } from "react";
    import {
      isPopupDismissed,
      dismissPopup,
      dismissForever,
      resetPopup,
    } from "@/src/services/popupService";

    interface UsePopupDismissalOptions {
      popupKey: string;
      /** If true, on mount the hook checks dismissal state and sets `visible` accordingly */
      autoCheck?: boolean;
    }

    interface UsePopupDismissalReturn {
      /** Whether the popup should be visible right now */
      visible: boolean;
      /** Call when user taps "Entendido" — dismisses per-session only */
      dismiss: () => void;
      /** Call when user checks "No mostrar de nuevo" then dismisses — permanent */
      dismissForeverFn: () => void;
      /** Reset both flags (dev/testing) */
      reset: () => void;
      /** Re-check AsyncStorage and update visible state (useful after background changes) */
      checkDismissal: () => Promise<void>;
    }
    ```

    Implementation:
    - `useState<boolean>(false)` for `visible`
    - `useEffect` on mount: if `autoCheck !== false`, call `checkDismissal()`
    - `checkDismissal`: await `isPopupDismissed(popupKey)` → set `visible` to NOT dismissed
    - `dismiss`: await `dismissPopup(popupKey)` → set `visible` to false
    - `dismissForeverFn`: await `dismissForever(popupKey)` → set `visible` to false
    - `reset`: await `resetPopup(popupKey)` → set `visible` to true

    All functions wrapped in `useCallback` with `[popupKey]` deps.
    `popupKey` is a string constant — does NOT need to be a dependency for effect (it's stable per component usage).
    Add JSDoc comment explaining the per-session vs permanent distinction per D-04/D-05.

    Create `src/services/activityService.ts` following the codebase service layer pattern (named exports, try/catch with console.error, Firestore subcollection access, Spanish comments).

    **Firestore Schema** (subcollection at `users/{uid}/activities/{docId}`):
    ```typescript
    interface ActivityEvent {
      type: "identify" | "water" | "achievement" | "mission";
      title: string;
      description: string;
      timestamp: Timestamp; // Firestore server timestamp
      iconType: "camera" | "water" | "award" | "sparkles";
      metadata?: {
        plantName?: string;
        plantId?: string;
        missionId?: string;
        achievementId?: string;
        xpEarned?: number;
      };
    }
    ```

    **Exported Functions:**

    1. `logActivity(userId: string, event: Omit<ActivityEvent, 'timestamp'>): Promise<string>` — Creates a new doc in `users/{userId}/activities` subcollection with `serverTimestamp()`. Returns the new doc ID. Must use `addDoc(collection(...), ...)` so Firestore auto-generates the ID. Set `timestamp` to `serverTimestamp()`.

    2. `getUserActivities(userId: string, limitCount?: number): Promise<ActivityEventWithId[]>` — Fetch all activities from `users/{userId}/activities`, ordered by `timestamp` descending. Default limit: 50. Returns array with `id` field from doc ID + `timestamp` converted to `Date` via `.toDate()`. Map Firestore `Timestamp` to `Date` using `safeParseDate` pattern from userService.

    3. `getRecentActivities(userId: string, days?: number): Promise<ActivityEventWithId[]>` — Same as getUserActivities but filtered to last N days (default 7). Uses Firestore `where("timestamp", ">=", ...)` with computed cutoff date.

    **Type to return** (extends ActivityEvent with id and Date):
    ```typescript
    interface ActivityEventWithId extends ActivityEvent {
      id: string;
      timestamp: Date; // Converted from Firestore Timestamp
    }
    ```

    **Mapping from ActivityEvent to ActivityFeed's ActivityData:**
    ```typescript
    function toActivityData(event: ActivityEventWithId): ActivityData {
      return {
        id: parseInt(event.id, 36) || 0, // hash doc ID to numeric
        type: event.type === "mission" ? "achievement" : event.type as any,
        title: event.title,
        description: event.description,
        time: formatRelativeTime(event.timestamp), // "Hace 2 horas", "Hace 3 días", etc.
        iconType: event.iconType as any,
      };
    }
    ```

    **Helper `formatRelativeTime(date: Date): string`:**
    - Calculate difference from now
    - < 1 min: "Ahora"
    - < 60 min: "Hace X min" (rounded)
    - < 24 hours: "Hace X h" (rounded)
    - < 7 days: "Hace X días" (rounded)
    - Otherwise: "dd/mm/aaaa" (date string)

    **Error handling:** All public functions wrapped in try/catch with `console.error("Error en activityService:", error)`. On error: logActivity re-throws; getUserActivities/getRecentActivities return empty array (fail-safe: show nothing rather than crash).

    **Import pattern:** Use `import { auth, db } from "@/src/config/firebase"` and `import { requireUserId } from "./userService"`.

    Create `src/hooks/useActivityFeed.ts` following the codebase hook pattern (named export, useCallback wrapping async calls, useState for state management, useFocusEffect for auto-refresh per useProfile.ts pattern).

    **Hook signature:**
    ```typescript
    export function useActivityFeed(options?: { limit?: number; days?: number; autoRefresh?: boolean }): {
      activities: ActivityData[];  // Already converted via toActivityData
      loading: boolean;
      error: string | null;
      reload: () => Promise<void>;
    }
    ```

    Defaults: `{ limit: 50, days: undefined, autoRefresh: true }`

    **Behavior:**
    - Gets userId from `getCurrentUserId()`
    - Initializes state: `{ activities: [], loading: !!userId, error: null }`
    - Async `loadActivities`: calls `getRecentActivities(uid, days)` if `days` is set, otherwise `getUserActivities(uid, limit)`, then maps via `toActivityData()`
    - If `autoRefresh !== false`, use `useFocusEffect` (from expo-router) to call `loadActivities` every time the screen gains focus — same pattern as useProfile.ts
    - `reload` returned so callers can manually trigger refresh

    **Error handling:** Sets `error` string on failure; `activities` stays as empty array or previous data on error (do NOT clear activities on error — keep showing stale data with error message preference, matching useProfile pattern).

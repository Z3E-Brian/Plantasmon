# Phase 3 — UI Design Contract

> Visual and interaction contract for Explore and Journal screens. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Manual React Native components (no shadcn) |
| Framework | React Native (Expo) + expo-router |
| Component library | Custom RN primitives (`View`, `Text`, `Pressable`, `TextInput`, `ScrollView`, `FlatList`) |
| Icon library | Emoji (primary) / `@expo/vector-icons` (available) |
| Font | System (no custom font loaded) |
| Styling pattern | `useThemedStyles()` hook with `StyleSheet.create()` per component |
| Navigation | expo-router `router.push()` with haptics (`expo-haptics`) |

### Established Patterns to Follow

- `ScreenWrapper` wrapping every screen
- `useThemedStyles("componentName")` for dark/light style separation
- Haptics on all press interactions (`Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Light)`)
- Spanish UI labels consistently
- `Pressable` with `android_ripple={{ color: "..." }}` for touch feedback
- `expo-image` for image rendering with `contentFit="cover"`
- ScrollView with `showsVerticalScrollIndicator={false}`

## Component Design

### PlantCard (Explore grid)

Inherited from `PlantCollection` component (`src/components/profile/PlantCollection.tsx`) with modifications for Explore context:

| Property | Value |
|----------|-------|
| Width | `(screenWidth - 52) / 2` |
| Background | `theme.colors.surface` |
| Border radius | `theme.radius.md` (16px) |
| Border | 1px `theme.colors.border` |
| Overflow | hidden |

**Card anatomy (top to bottom):**
1. **Image container** — 112px height, with backgroundColor fallback `theme.colors.primarySoft`
2. **Health tag** — absolute positioned bottom-left, small pill with health status text
3. **Info section** — 10px padding, contains:
   - Plant name: 12px weight 700, `textPrimary`, single line (ellipsis)
   - Scientific name: 10px italic, `textSecondary`, single line
   - Care row: water drops (💧 1-3 active) + sun icon (☀️ opacity) + days owned "Xd"

**Explore-specific additions to PlantCard:**
- **Rarity badge** — top-right corner of image (absolute positioned). Pill badge: "Común", "Poco Común", "Rara", "Legendaria" with corresponding rarity colors
- **Difficulty indicator** — text badge or visual indicator near the name
- **Difference from UserPlant cards:** No favorite button (this is a catalog, not user's collection). No edit/remove actions.

### SearchBar (Explore)

| Property | Value |
|----------|-------|
| Container | `flexDirection: "row"`, `alignItems: "center"`, rounded (16px) |
| Background | `theme.colors.surfaceMuted` |
| Border | 1px `theme.colors.border` |
| Padding | horizontal: 12px |
| Icon | 🔍 emoji, 14px, `textTertiary` color |
| Input | `flex: 1`, height 40px, 14px font, `textPrimary` color |
| Placeholder | "Buscar plantas..." in Spanish, `textTertiary` color |

**Behavior:**
- Debounced text input (300ms debounce)
- Filter by `commonName` and `scientificName` (case-insensitive substring match)
- Clear button (X) appears when text is entered
- Loading state while filtering (no network call — filter local data)

### Journal Cards (4 card types)

All cards share these base styles (inherited from `LastIdentified`, `UserProgress`, `DailyMissions`, `RecentAchievement` patterns):

| Property | Value |
|----------|-------|
| Horizontal margin | 20px |
| Bottom margin | 18px (`theme.spacing.lg`) |
| Padding | 18px (`theme.spacing.lg`) |
| Background | `theme.colors.surface` (except achievement which uses `primarySoft`) |
| Border radius | `theme.radius.md` (16px) |

**Card 1 — "Tus Plantas Analizadas"**
- Header row: "🌱 Tus Plantas Analizadas" title + count badge
- Horizontal row of 3-4 plant thumbnails (48x48px, rounded 12px) + "+N más"
- "Ver todas →" footer link
- Empty state: "Aún no has identificado plantas. ¡Usa la cámara para empezar!"

**Card 2 — "Objetivos / Misiones"**
- Reuse `DailyMissions` component patterns directly
- Title: "🎯 Objetivos de Hoy"
- List of missions with ○/✓ indicators and XP badges
- No empty state needed (always show default missions)

**Card 3 — "Tu Progreso"**
- Reuse `UserProgress` component patterns directly
- Title: "📊 Tu Progreso"
- Progress bar + level badge + XP text
- No empty state needed (always show stats)

**Card 4 — "Logros Recientes"**
- Conditional — only render if user has at least one earned achievement
- Reuse `RecentAchievement` pattern: standout card with `primarySoft` bg and `primary` border
- Icon (48px) + achievement name (18px weight 600) + description
- Navigate to profile/achievements on press

## Accessibility

### Touch Targets
- All `Pressable` elements: minimum 44x44px (search clear button, health/filter toggles)
- Card touch area: full card surface (pressable background)
- Bottom nav items: inherit existing pattern (already >44px)

### Color Contrast
- Body text (14px #e8f5e9 on #0d3320 bg): ~9.5:1 contrast ratio (passes WCAG AAA)
- Secondary text (14px #BECCBE on #0d3320 bg): ~8:1 contrast ratio (passes WCAG AA+)
- Tertiary text/placeholders (14px #95A796 on #0d3320 bg): ~5.5:1 (passes WCAG AA at 14px)
- Primary (green #40916c) on dark backgrounds: ~4.5:1 (WCAG AA)
- Destructive (#ef4444) on dark backgrounds: ~4.6:1 (WCAG AA)

### Semantic Structure
- Screen titles use `<Text>` with appropriate weight/size (no heading level semantics — React Native doesn't support heading landmarks natively)

### Motion
- Haptic feedback on all press actions per existing pattern
- No auto-animating content (search debounce is the only async interaction)
- Pull-to-refresh is default ScrollView behavior — keep as-is

### Focus
- Search TextInput auto-focus is NOT recommended (users may want to browse first)
- Clear search button: accessible via onPress

## Interaction Patterns

### Explore Screen

| Interaction | Behavior |
|-------------|----------|
| **Search input** | Live text filter with 300ms debounce. Filters by plant name (common + scientific). Case-insensitive. No network call. |
| **Clear search** | X button appears when text is non-empty. Clears input + resets filtered results. |
| **Card tap** | `Haptics.impactAsync(.Light)` → `router.push(\`/plant/${plantId}\`)`. If target route doesn't exist yet, show placeholder screen or Toast "Próximamente". |
| **Scroll** | Standard vertical scroll via FlatList. `showsVerticalScrollIndicator={false}`. Bottom safe area padding. |
| **Pull-to-refresh** | RefreshControl on FlatList re-fetches plants from Firestore. Show refreshing indicator. |

### Journal Screen

| Interaction | Behavior |
|-------------|----------|
| **Scroll** | Standard vertical scroll via ScrollView. `showsVerticalScrollIndicator={false}`. Bottom safe area + BottomNav padding. |
| **Card tap (Plants)** | "Ver todas" → `router.push("/profile")` (existing plant collection tab). |
| **Card tap (Missions)** | Mission row tap → visual toggle (no action needed unless deferred to future phase). Use haptics. |
| **Card tap (Progress)** | No tap action — static display. |
| **Card tap (Achievements)** | Card tap → `router.push("/profile")` to achievements section. Use haptics. |
| **Pull-to-refresh** | RefreshControl on ScrollView re-fetches user data (plants, achievements, missions). |

### Shared Patterns

| Pattern | Implementation |
|---------|---------------|
| **Haptics** | All `onPress` handlers call `Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Light)` |
| **Ripple** | Android ripple on Pressable: `android_ripple={{ color: "rgba(64,145,108,0.15)" }}` |
| **Navigation** | `expo-router` with `router.push()` (not replace) for back-navigation support |
| **Data loading** | `useEffect` + async fetch pattern. Use `useState` for loading/error/data. |
| **Toast errors** | `react-native-toast-message` for transient error feedback |

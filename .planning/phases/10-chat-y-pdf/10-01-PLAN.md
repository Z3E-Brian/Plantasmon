### Chat Backend API Reference (from analysis)

The backend is Python FastAPI (not Node.js). Key endpoints:

| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | `/health` | Health check |
| POST | `/api/chat/join` | Register nickname, get JWT token |
| POST | `/api/chat/logout` | Revoke JWT token |
| GET | `/api/chat/users` | List online users |
| GET | `/api/chat/messages` | Get group chat history |
| POST | `/api/chat/messages` | Send a message (group or DM) |
| GET | `/api/chat/messages/dm/{other_id}` | Get DM history |
| PUT | `/api/chat/users/me/public-key` | Register public key for encryption |
| POST | `/api/chat/media/upload` | Upload file to Cloudinary |
| WS | `/ws/{token}` | WebSocket for real-time messaging |

**WebSocket protocol:** All messages are JSON. Client sends: `group_message`, `dm`, `mark_read`, `typing`, `stop_typing`, `ping`. Server responds with: `group_message`, `dm`, `message_seen`, `message_expired`, `typing`, `user_joined`, `user_left`, `users_list`, `group_history`, `group_key`, `pong`, `error`.

| Boundary | Description |
|----------|-------------|
| Client → Chat Backend API | HTTP requests with JWT tokens; untrusted client input crosses here |
| Client → Chat WebSocket | Real-time messaging; message content crosses here |
| Backend → Cloudinary | Media uploads with API credentials; internal boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Spoofing | POST /api/chat/join | mitigate | JWT tokens with HS256 signature; token validation in `decode_token()` |
| T-10-02 | Tampering | WebSocket messages | accept | End-to-end encryption is optional (handled client-side); backend validates content length (1000 char max) |
| T-10-03 | Information Disclosure | .env file | mitigate | .gitignore excludes .env; production env vars set via Render dashboard, never committed |
| T-10-04 | Denial of Service | WebSocket connections | accept | In-memory only (no persistence); connection count limited by Render free tier constraints |
</threat_model>

<action>
Create src/hooks/useAchievementUnlock.ts that:

1. Exports a function checkAndUnlockAchievements(userId: string) that:
   - Reads user's achievements from Firestore (userAchievements array on user doc)
   - Reads achievements collection definitions
   - For each achievement NOT yet unlocked, checks if its requirement is met
   - If met, adds to user's unlockedAchievements array on the user doc
   - Returns an array of newly unlocked achievements (for toast notification)

2. Currently checks these requirement types:
   - "identifications": getUserProfile(userId) -> plantsOwned field, compare to count
   - "streak": getUserProfile(userId) -> streak field, compare to count
   - "species": needs species count from userPlants (count distinct scientificNames)
   - "waterings": getUserProfile(userId) -> could use existing streak data or totalWaterings
   - "days_active": calculated from joinDate
   - For other types (rare_find, daily_missions, customization, fun_facts): stub with console.warn (not implemented yet)

3. The unlock data is stored on user doc as:
   ```
   unlockedAchievements: [
     {
       id: "achievement_id",
       unlockedAt: ISO date string,
       progress: number (current progress toward requirement)
     }
   ]
   ```

4. Shows console.log for each newly unlocked achievement
5. Returns Promise<string[]> (array of newly unlocked achievement IDs)
</action>

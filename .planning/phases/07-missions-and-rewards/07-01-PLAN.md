    2. **Types/exports:**
    ```typescript
    export type MissionType = "daily" | "weekly";

    export interface AssignedMission {
      id: string;
      type: MissionType;
      progress: number;       // current progress (e.g., 2)
      target: number;         // required to complete (e.g., 3)
      completed: boolean;
      claimed: boolean;       // XP claimed?
      claimedAt?: string;     // ISO date when claimed
    }

    export interface MissionProgress {
      existing: { id: string; progress: number }[];
      currentValue: number;
    }
    ```

    3. **Core functions:**

    **seedMissions()** — Check if `missions` collection has data. If empty, iterate over DAILY_MISSIONS + WEEKLY_MISSIONS and `setDoc(doc(db, "missions", m.id), { ...m, type: "daily"|"weekly" }, { merge: true })`. Follow the exact seedAchievements() pattern from userAchievementsService.ts.

    **getMissionDefinitions()** — Fetch all docs from `missions` collection. Return array of MissionDefinition with `type` field.

    **assignDailyMissions(userId, allMissions)** — Pick 5 random daily missions from the daily pool. Deterministic selection based on day number (so user sees same 5 all day). Store assigned IDs in user doc under `missions.assignedDailyIds`. Set `missions.lastDailyRefresh` to today's ISO date.

    **assignWeeklyMissions(userId, allMissions)** — Pick 2 random weekly missions. Deterministic based on week number. Store in `missions.assignedWeeklyIds`. Set `missions.lastWeeklyRefresh` to current week number.

    **needsDailyRefresh(userDoc)** — Returns true if `userDoc.missions.lastDailyRefresh` is not today (compare `toDateStr(new Date())` with stored date).

    **needsWeeklyRefresh(userDoc)** — Returns true if `userDoc.missions.lastWeeklyRefresh !== getWeekNumber(new Date())`.

    **getWeekNumber(date)** — Helper: `Math.ceil(((date - new Date(date.getFullYear(), 0, 1)) / 86400000 + new Date(date.getFullYear(), 0, 1).getDay() + 1) / 7)`

    **getUserMissions(userId?)** — Fetch user doc, get `missionProgress[]`, assignedDailyIds/assignedWeeklyIds, resolve definitions from missions collection. Return `{ daily: AssignedMission[], weekly: AssignedMission[], needsRefresh: { daily: boolean, weekly: boolean } }`.

    **updateMissionProgress(userId, missionId, increment = 1)** — Find the mission in user doc's `missionProgress`, increment progress. If progress >= target, set completed = true. Use `arrayUnion` or direct doc update with map merge.

    **claimMissionReward(userId, missionId)** — Find mission in user doc's `missionProgress`, set `claimed = true`, `claimedAt = new Date().toISOString()`. Add XP to `stats.xp`. Return the XP amount.

    **getExpiredMissions(userId)** — Return missions from the PREVIOUS day's assignment that are completed but not claimed. Used for grace period display.

    4. Add `toDateStr` helper (same pattern as in userService.ts):
    ```typescript
    function toDateStr(d: Date): string {
      return `${d.getFullYear()}-${String(d.getMonth()+1).padStart(2,"0")}-${String(d.getDate()).padStart(2,"0")}`;
    }
    ```

    5. Errors: wrap each function in try/catch, log with `console.error`, re-throw meaningful Spanish error messages.
